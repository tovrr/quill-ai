Title: Generate API docs and OpenAPI spec

## Purpose

Help an agent produce documentation for HTTP API routes and a minimal OpenAPI/Swagger spec suitable for codegen and docs.

## Inputs

- `route`: path or file containing the API handler (e.g., `src/app/api/chat/route.ts`).
- `format`: `openapi` or `raml` (default `openapi`).

## Tasks

1. Parse handler input/output shapes and infer request/response JSON schemas.
2. Produce an OpenAPI `paths` entry and component schemas for shared types.
3. Generate a docs markdown file with examples and suggested usage.

## Output (JSON)

{
"openapi": {"openapi":"3.1.0","paths":{...},"components":{...}},
"docsFile": {"path":"docs/api/chat.md","content":"..."}
}
