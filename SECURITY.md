# Security Policy

## Supported Versions

Security fixes are prioritized on the `main` branch.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Preferred process:

1. Open a private GitHub security advisory for this repository.
2. Include reproduction details, impact, and suggested remediation.

If private advisory flow is unavailable, open an issue with minimal details and request a private contact channel.

## Response Expectations

- Initial triage target: within 72 hours
- Status updates during investigation
- Coordinated disclosure after a fix is available
